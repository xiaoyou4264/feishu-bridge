# Quick Task 260407-p25: Card Streaming — Split Content & Timer Elements

## What Changed

Split the streaming card from a single element into two independent elements to fix two problems:
1. Timer updates breaking the typing animation (non-prefix content changes)
2. Timer only refreshing every 3 seconds (felt unresponsive)

## Files Modified

| File | Change |
|------|--------|
| `src/card_streaming.py` | Refactored to dual-element architecture: `md_stream` (content) + `md_timer` (timer). Added `_timer_loop()` at 1s interval, `_put_element()` generic PUT method, removed timer from `_build_display_text()` |
| `src/cards.py` | Added `md_timer` element + `<hr>` divider to streaming card template |
| `tests/test_card_streaming.py` | Updated tests for dual-element behavior: timer targeting, shared sequence, display text without timer, start creates both tasks, finalize updates both elements |

## Architecture

```
Before: Single element (md_stream)
  [tool blocks + text + typing indicator + --- + timer]
  → Every timer change = full content replacement = jarring re-render

After: Two independent elements
  md_stream: [tool blocks + text + typing indicator]
  ---  (hr element)
  md_timer: [timer status line]
  → Content only grows (prefix-extension = smooth typing animation)
  → Timer updates independently at 1s (no flicker on main content)
```

## Key Design Decisions

- **Shared sequence counter** — CardKit sequences are per-card, not per-element. Both elements share `_sequence`
- **Timer interval: 1 second** — independent updates make this cheap (tiny payload, no content interference)
- **Typing indicator stays in md_stream** — "正在思考中..." / "正在输入..." are part of the content flow
- **`_put_element(element_id, content)`** — generic method replaces the old `_put_content()` (kept as alias for backward compat)
