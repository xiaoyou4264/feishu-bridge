# P25: Card Streaming — Split Content & Timer Elements

## Problem

The `CardStreamingManager` embeds the timer line inside the main content element (`md_stream`). Every timer update changes the middle/end of the content string, breaking the CardKit prefix-extension detection. This causes jarring full-replacement renders instead of smooth typing animation.

Additionally, the timer only updates every 3 seconds (`_TIMER_INTERVAL = 3`) to reduce re-renders, making it feel unresponsive.

## Solution

Split the card into **two independent elements**:
1. **`md_stream`** — main content (tool blocks + text). Only grows via prefix-extension → smooth typing animation.
2. **`md_timer`** — timer status below a divider. Updated independently at 1-second intervals.

This ensures main content PUTs are always prefix-extensions (typing effect works), while timer updates are small independent PUTs that don't interfere.

Import `TIMER_ELEMENT_ID` from `src.card_streaming` alongside existing `STREAMING_ELEMENT_ID` import.

### Task 2: Refactor CardStreamingManager for dual-element updates (`src/card_streaming.py`)

**Files:** `src/card_streaming.py`

Changes:
1. Add constant `TIMER_ELEMENT_ID = "md_timer"` 
2. New `_put_timer(content: str)` method — PUT to timer element (same pattern as `_put_content` but targets `md_timer`, uses own sequence counter or shared)
3. New `_timer_loop()` coroutine — runs at 1-second interval, PUTs timer element only
4. `_flush_loop()` — remove all timer logic (`_TIMER_INTERVAL`, `last_timer_sec`, `timer_due`). Only PUT when content body changes. Remove timer from `_build_display_text()`.
5. `_build_display_text()` — remove timer line and `---` divider from output. Only returns tool blocks + text + optional typing indicator.
6. `start()` — launch both `_flush_task` and `_timer_task`
7. `finalize()` — cancel both tasks. PUT final content to `md_stream`, PUT final timer to `md_timer`, then close streaming mode.

Key design decisions:
- **Single sequence counter** shared between both elements (CardKit sequences are per-card, not per-element)
- Timer loop interval: **1 second** (down from 3s) — independent updates make this cheap
- Typing indicator (`正在输入...` / `正在思考中...`) stays in `md_stream` content, NOT in timer element
- Timer element format: just the backtick-wrapped status line, no extra formatting
